# Claude Code 工作流配置

这是一个完整的产品开发工作流配置，集成了产品经理、设计师和开发者三个专业角色。

## 工作流程概述

```
用户需求 → 产品经理 → 设计师 → 开发者 → 最终产品
     ↓         ↓         ↓         ↓
   收集需求 → 生成PRD → 设计规范 → 代码实现
```

## 指令映射

### /PRD - 生成产品需求文档
**触发**: 当用户描述产品想法或需求时
**执行**: 激活产品经理专家
**输出**: PRD.md - 包含详细的产品需求规格
**后续**: 为设计师提供输入依据

### /设计 - 生成设计规范
**触发**: 用户输入 `/设计` 指令，或自动检测到设计相关需求
**执行**: 激活设计师专家
**输入**: 读取 PRD.md 文档
**输出**: DESIGN_SPEC.md - 包含UI/UX设计规范
**后续**: 为开发者提供设计指导

### /开发 - 生成代码实现
**触发**: 用户输入 `/开发` 指令，或自动检测到开发需求
**执行**: 激活开发者专家
**输入**: 读取 PRD.md 和 DESIGN_SPEC.md 文档
**输出**: 完整的代码文件和项目结构
**后续**: 生成可运行的产品

## 工作流控制规则

### 阶段顺序要求
1. **必须先有 PRD**: 设计师和开发者都需要读取 PRD.md 作为基础输入
2. **设计依赖需求**: 设计师必须在 PRD 生成后才能创建设计规范
3. **开发依赖设计**: 开发者需要同时参考 PRD 和设计规范

### 自动流程引导
当用户描述产品想法时，系统会：
1. 自动激活产品经理收集需求
2. 提示用户输入 `/PRD` 生成文档
3. PRD 生成后提示输入 `/设计`
4. 设计完成后提示输入 `/开发`

### 文档传递机制
- **PRD.md**: 产品需求文档，作为整个流程的起点
- **DESIGN_SPEC.md**: 设计规范文档，从 PRD 派生
- **代码文件**: 最终实现，同时依赖 PRD 和设计规范

## 使用指南

### 新项目启动
1. 向 Claude 描述你的产品想法
2. 系统会引导你完成需求收集
3. 依次执行 `/PRD` → `/设计` → `/开发`
4. 每个阶段都会生成相应的文档和代码

### 阶段验证
- **PRD阶段**: 检查 PRD.md 是否包含完整的功能需求
- **设计阶段**: 检查 DESIGN_SPEC.md 是否有详细的UI规范
- **开发阶段**: 检查代码是否完整可运行

### 迭代改进
如果需要修改，可以：
1. 更新 PRD.md 然后重新执行 `/设计` 和 `/开发`
2. 单独更新设计规范然后重新执行 `/开发`
3. 直接修改代码实现

## 技能独立性

每个技能都保持独立，职责清晰：
- **产品经理**: 只负责需求分析和文档化
- **设计师**: 只负责UI/UX设计规范
- **开发者**: 只负责技术实现

通过主控文件实现协作，确保流程的连贯性和可复用性。

## 自定义扩展

你可以根据需要添加更多技能：
- 测试工程师：代码审查和测试
- 技术文档：API文档生成
- DevOps：部署和运维

只需要在 skills/ 目录下添加新的技能目录和 SKILL.md 文件，然后在 CLAUDE.md 中配置相应的指令映射即可。
